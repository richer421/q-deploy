# 核心语义

## 项目

- **名称**: q-deploy
- **定位**: 下一代服务发布平台，AI 与云原生应用
- **领域**: 云原生应用交付 / DevOps / 服务发布

## 系统边界

做什么：
- YAML 渲染（基于服务配置和模板引擎生成最终的部署配置）
- 发布操作（创建新版本部署）
- 回滚操作（回退到历史版本）
- 更新操作（配置变更，包产物不变）

不做什么：
- 服务配置管理（由元数据管理中心负责）
- CI 构建（由外部 CI 系统负责）
- 源码管理（不涉及代码仓库的版本控制）
- 监控告警（由独立可观测性系统负责）

## 核心业务概念

| 术语 | 含义 |
|------|------|
| 发布 | 完整的部署操作，新产物 + 新工作负载 YAML + 新配套资源 YAML |
| 回滚 | 回退到历史版本，恢复工作负载 YAML + 配套资源 YAML |
| 更新 | 配套资源变更（ConfigMap/Secret/Service 等），工作负载和产物不变 |
| 服务配置 | 服务的部署参数定义，由元数据管理中心维护，本平台作为输入消费 |
| 渲染器 | 将服务配置转换为最终 YAML 的模板引擎（helm/kustomize/go_template） |
| 工作引擎 | 执行实际发布/回滚/更新操作的执行器（kubernetes/docker/ssh） |
| 包产物 | CI 构建输出的制品，类型包括：image（容器镜像）、binary（二进制）、archive（压缩包） |
