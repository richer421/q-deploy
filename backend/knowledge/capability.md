# 核心业务能力

描述系统解决的业务问题和核心能力边界。

## 设计原则

- 只描述"做什么"，不描述"怎么做"
- 接口细节由 Swagger 文档维护，此处不重复
- 新增模块时，说明业务场景而非 API 列表

## 能力清单

### 渲染引擎

将服务配置（来自元数据管理中心）转换为可执行的 YAML 配置。

- 多种渲染器支持（Helm、Kustomize、自定义模板等）
- 模板变量注入与替换
- 渲染结果预览与校验

### 发布管理

核心发布能力，支持多种发布模式。

- **发布**：完整部署，包含新包产物 + 新配置
- **更新**：配置变更型发布，包产物不变
- **回滚**：回退到历史版本
- 发布单管理与状态追踪

### MCP Server

独立 MCP Server，供 Claude 调用：

- `q-deploy mcp` — 启动 MCP Server（stdio 模式）
- 工具：
  - `call_api` — 调用 API
  - `read_logs` — 读取日志文件
